Event Management API

API Endpoints

This API allows you to manage events with the following endpoints:

  • POST /events - Create a new event

    Description: Adds a new event to the database with provided name, date, and location.

  • GET /events - Retrieve all events

    Description: Fetches a list of all events in the database.

  • GET /events/:id - Retrieve event by event ID

    Description: Fetches the details of a specific event by its ID.

  • GET /events/name/:name - Retrieve event by event Name

    Description: Fetches events that match a specific name.

  • PUT /events/:id - Update event by event ID

    Description: Updates the details of a specific event identified by its ID.

  • DELETE /events/:id - Delete event by event ID

    Description: Deletes a specific event identified by its ID.

  • DELETE /events - Delete all events

    Description: Deletes all events from the database.

Postman Test Example

To test the "Create a new event" function, send a POST request to http://localhost:3000/events with the following JSON body:

        {
          "name": "Test Event",
          "description": "Description of Event",
          "startdate": "2024-05-20 08:00",
          "enddate": "2024-05-20 17:00"
        }

The response should include a 201 status code and the details of the created event.
